Post [ Help Index ]

A Post is the position on which the candidate can apply for a job. The details of the Post are to be decided by the company. It reflects all the details of the position for which the company requires an eligible candidate.

The post details include the following :

 Post Name This is the name of the post in the company and must be unique. It is based on the conventions of Name Field and should not exceed more than 20 characters. It can also be in the form of codes or numbers except the first alphabet.
 Description  This is the description of the post of information purpose. The total characters should not exceed more than 500.
 Minimum  Percent This option specifies the minimum average percentage of the tests for this post the candidate should obtain to qualify for this post. The value can range anything between 0 to 100. If the candidate fails to achieve this specified minimum percent he will be marked red and put into the failures list in the Result View.
 Qualification  The Qualification indicates the minimum required qualification for this post or job position. This totally depends on company and its managament.
 Experience  This specifies the minimum no of years of working experience required for this post or job position. It depends on the company and can be anything between 0 years to more than 10 years.
 Branch The Branch specifies the branch of qualification and experience the candidate should persist depending on the requirement of the company.
 Age Limit  The age limit denotes the age restriction of the candidate applying for this post. If there is no age restriction the selection box is set to 'no'. If there is an age restriction then the maximum no of years the age of the candidate should be specified which can be anything between 18 to 60 years.
 Vacancy  This option sets the amount of vacant positions of this particular post in the company. When the candidates are recruited in the View Selected Candidates the vacancy decreases automatically while it increases in the View Employees when the candidates are deleted. The number of vacant seats can range to anything between 0 to 100000. It is manditory for the management to set this field to at least one so that the candidates can apply for the post.
 Post  Categories 

This option enables the company to decide the Categories ie. the subjects for which the test is to be conducted. It may vary from post to post and needs of the company. The company management can make appropriate selection of the Categories depending on the skill sets the company expects the applying candidate should persist. If there are no Categories specified under this option no candidate can apply for this post by default. It is important to know that if the candidate fails in anyone of the specified category tests he/she will be marked red and put into the failures list in the Result View.
